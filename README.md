# RouteHelper — Windows 路由管理工具

## 场景说明

在公司/内网环境中，你可能会遇到这样的情况：

```
  有线网卡（接公司内网）    ← 访问内网服务器时需要走这里
  ─────────────────────
  WiFi（接外网）            ← 浏览网页、聊天等走这里
```

Windows 默认只有一个**默认路由**（`0.0.0.0/0`），通常先连上的网卡独占默认路由，后连上的抢不走，导致要么全走有线（WiFi 没网），要么全走 WiFi（内网访问不了）。

**解决方法**：保留 WiFi 的默认路由，再手动添加一条指向**有线网卡网关**的内网段路由，让内网流量走有线，其余流量走 WiFi。

右键点击 `RouteHelper_v3.ps1` → **"使用 PowerShell 运行"**（以管理员身份）。

或者手动以管理员身份打开 PowerShell，执行：


> ⚠ **必须以管理员身份运行**，因为添加/删除路由需要管理员权限。

## v3 功能介绍

```
==========================================
       RouteHelper — 路由管理工具
==========================================

  1.   添加路由
        （为指定网卡添加临时路由，退出自动清理）

  2.   查看路由
        （按条件筛选当前 IPv4 路由表）

  3.   删除路由
        （手动删除一条路由）

  4.   清理脚本添加的临时路由
        （删除本次运行中通过此脚本添加的所有路由）

  5.   退出
```

### 功能 1：添加路由

支持的特性：
- 自动列出所有活跃的网卡及其网关，**手动选择**走哪张网卡
- 输入目标网段 + 子网掩码
- 可选设置**跃点数（Metric）**，优先级灵活调整
- 可选添加**备注**，方便区分用途
- 添加前回显完整信息供确认
- 脚本自动记录，退出时可一键清理

### 功能 2：查看路由

支持多种筛选方式：
- 所有 IPv4 路由
- 仅默认路由（`0.0.0.0/0`）
- 按指定网卡查看
- 按目标网段搜索
- 仅非默认路由（即手动添加的那些）

### 功能 3：删除路由

- 先列出当前所有非默认路由供参考
- 输入目标网段即可删除
- 支持 CIDR 格式输入（如 `10.99.0.0/16`），自动去掉掩码位

### 功能 4：清理临时路由

- 列出本次脚本添加的所有路由
- 确认后一键全部删除
- 退出脚本时也会询问是否清理

## v2 功能介绍（单次添加模式）

v2 适合"快速添加一条路由，用完删除"的简单场景：

```
检测到可用网卡 → 选择序号 → 输入网段 → 确认添加 → 按任意键删除
```

- 自动检测有线网卡，找不到则列出所有网卡供选
- `try-finally` 确保 Ctrl+C 也不会残留路由
- `route add` 自动带上接口索引（`IF` 参数），多网卡更准确
- 输入格式校验 + 添加前回显确认

## 典型示例

```text
内网服务器 IP： 10.99.1.100
目标网段：      10.99.0.0
子网掩码：      255.255.0.0
有线网关：      192.168.1.1

v3 操作步骤：
1. 运行脚本 → 选 1 添加路由
2. 选择有线网卡（序号）
3. 输入目标网段：10.99.0.0
4. 输入子网掩码：255.255.0.0
5. 确认添加
6. 正常访问内网资源
7. 回到菜单选 4 或直接退出时清理
```

## 常见问题

### 找不到有线网卡？

脚本会自动列出当前所有默认路由供你选择：

```
[0] Realtek PCIe GbE Family Controller    → 网关 192.168.1.1
[1] Wi-Fi                                 → 网关 10.0.0.1
请选择序号（直接回车默认 0）：
```

### 路由添加失败？

- 检查目标网段是否与现有路由冲突（脚本功能 2 可查看）
- 检查网关 IP 是否正确
- 确认网卡物理连接正常
- 确认以管理员身份运行

### 路由没有生效？

- 用功能 2 查看路由表中是否已有目标网段的条目
- 可能有其他路由策略（如 VPN 客户端）产生冲突
- 先 ping 网关确认连通性

### 怎么手动查看当前路由表？

```cmd
route print
```

## 技术原理

Windows 路由表按**最长前缀匹配**规则工作：

- `0.0.0.0/0` — 默认路由，匹配所有地址（WiFi 的默认路由）
- `10.99.0.0/16` — 更具体的路由，只匹配 `10.99.x.x`（有线网卡）

当访问 `10.99.1.100` 时，系统优先匹配更具体的 `10.99.0.0/16`，所以流量走有线网卡。
当访问 `114.114.114.114` 时，没有更具体的匹配，走默认路由的 WiFi。

# RouteHelper — 内网路由管理脚本

## 场景说明

在公司/内网环境中，你可能会遇到这样的情况：

```
  有线网卡（接公司内网）    ← 访问内网服务器时需要走这里
  ─────────────────────
  WiFi（接外网）            ← 浏览网页、聊天等走这里
```

Windows 默认只有一个**默认路由**（`0.0.0.0/0`），通常先连上的网卡独占默认路由，后连上的抢不走，导致要么全走有线（WiFi 没网），要么全走 WiFi（内网访问不了）。

**解决方法**：保留 WiFi 的默认路由，再手动添加一条指向**有线网卡网关**的内网段路由，让内网流量走有线，其余流量走 WiFi。

本脚本就是干这个事的——**自动检测有线网卡 → 添加临时内网路由 → 用完自动清理**。

## 使用方法

### 1. 运行脚本

右键点击 `RouteHelper_v2.ps1` → **"使用 PowerShell 运行"**，或在 PowerShell 中执行：

```powershell
powershell -ExecutionPolicy Bypass -File "G:\test\路由\RouteHelper_v2.ps1"
```

> ⚠ **必须以管理员身份运行**，因为添加/删除路由需要管理员权限。

### 2. 按提示操作

```
检测到有线网卡：[以太网]  网关：192.168.1.1
请输入要访问的内网目标网段（例如 10.99.0.0）：
请输入子网掩码（直接回车则使用 255.255.255.0）：
```

### 3. 使用内网资源

路由添加成功后，保持 PowerShell 窗口打开即可正常访问内网资源。

### 4. 用完清理

回到 PowerShell 窗口按**任意键**，脚本自动删除刚才添加的路由。即使按 `Ctrl+C` 中断也会自动清理，不会留下残留路由。

```text
内网服务器 IP： 10.99.1.100

操作步骤：
1. 运行脚本
2. 输入目标网段：10.99.0.0

4. 确认添加
5. 正常访问内网资源
6. 用完按任意键清理
```

## 常见问题

### 找不到有线网卡？

如果有线网卡名称不是"以太网"、"Ethernet"、"有线"、"LAN"等常见名称，脚本会自动列出当前所有默认路由供你选择：

```
[0] Realtek PCIe GbE Family Controller    → 网关 192.168.1.1
[1] Wi-Fi                                 → 网关 10.0.0.1
请选择序号（直接回车默认 0）：
```

### 路由添加失败？

- 检查目标网段是否与现有路由冲突（可用 `route print` 查看）
- 检查有线网卡的网关 IP 是否正确
- 确认有线网卡物理连接正常
- 确认以管理员身份运行

### 路由没有生效？

- 运行 `route print` 查看路由表中是否已有目标网段的条目
- 可能有其他路由策略（如 VPN 客户端）产生冲突
- 先 ping 有线网关确认连通性

### 怎么查看当前路由表？

```cmd
route print
```

## 技术原理

Windows 路由表按**最长前缀匹配**规则工作：

- `0.0.0.0/0` — 默认路由，匹配所有地址（WiFi 的默认路由）
- `10.99.0.0/16` — 更具体的路由，只匹配 `10.99.x.x`（有线网卡）

当访问 `10.99.1.100` 时，系统优先匹配更具体的 `10.99.0.0/16`，所以流量走有线网卡。
当访问 `114.114.114.114` 时，没有更具体的匹配，走默认路由的 WiFi。

### 两种版本的核心区别

| 对比项 | 初版 (v1) | 改进版 (v2) |
|--------|-----------|-------------|
| 执行命令方式 | `Invoke-Expression`（有风险） | 直接调用 `route` 命令 |
| 异常清理 | Ctrl+C 会残留路由 | `try-finally` 兜底清理 |
| 多网卡选择 | 硬匹名称，找不到就手输 | 列出所有路由供选择 |
| 输入校验 | 无 | 正则校验 + 转数字兼容 |
| 接口索引 | 不带 IF 参数 | 自动带上接口索引 |
| 确认步骤 | 无 | 添加前回显确认 |
